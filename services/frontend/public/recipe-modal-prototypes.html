Design 1 — Long Scroll (Baseline)

Faithful reproduction of the current design — mobile-style full-scroll. All fields visible in one continuous flow with a sticky header.

  • Everything visible in one pass; no context switching
  • Familiar to current users; no learning curve
  • Simplest implementation and mental model
  • Very long form; nutrition buried far below the fold
  • No hierarchy between required vs optional fields
  • Overwhelming for new users

Design 2 — Tabbed Sections ★ Top Pick

Segmented control splits the form into Basics / Nutrition / Content — each a focused screen. Feels like a native iOS flow.

  • Focused — each screen is short and scannable
  • Nutrition is opt-in without being hidden
  • Feels native iOS; familiar interaction pattern
  • Requires mental model of what's on each tab
  • More taps to complete full entry
  • Easy to forget a tab entirely

Design 3 — Two-Column Split

Fixed left sidebar (image + meta stats), scrollable right main area (title, content, nutrition). Great for desktop workflows.

  • Image and meta always visible while editing content
  • Dense information layout for power users
  • Great for desktop; feels like a pro tool
  • Cramped on narrow viewports; sidebar competes visually
  • Two separate scroll areas can disorient users
  • Nutrition still at the bottom of right column

Design 4 — Step Wizard ★ Top Pick

5-step guided flow. Each step shows one focused group of fields with Back/Next navigation and a dot progress indicator.

  • Lowest per-step cognitive load; very focused
  • Forces complete entry; progress visible at a glance
  • Great for first-timers; clear sense of completion
  • Multi-step friction for experienced users
  • Can't see everything at once; revisiting requires navigation
  • 5 steps may feel excessive for a simple recipe

Design 5 — Live Preview Panel

Left half = scrollable form; right half = sticky live recipe card preview updating in real-time. Instant WYSIWYG feedback.

  • Instant WYSIWYG feedback; feels like a pro tool
  • Motivates completion — preview fills up as you type
  • Elegant and impressive for desktop users
  • Requires wide viewport; left form is still long
  • Complex UI risks feeling busy or overwhelming
  • Preview may not reflect actual published appearance
